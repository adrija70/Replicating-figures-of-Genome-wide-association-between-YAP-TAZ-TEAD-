### 0. What problem this code is solving (big picture)

You have **ChIP-seq peak files** for:

- YAP
- TAZ
- TEAD4

Each file contains **genomic intervals** (chromosome + start + end).

What is (chromosome + start + end)?
A peak is defined by the **chromosome**, the **base-pair position where enrichment begins (start)**, and the **base-pair position where enrichment ends (end)**.

Your goals here are:

1. Read these peak files into R
2. Treat them as genomic objects
3. Ask: *Which YAP peaks overlap TAZ peaks?*
4. Count overlaps
5. Draw a **Venn diagram** of YAP vs TAZ peaks

## 1. Reading peak files into R

```r
TAZ_peaks<- import(here("fastq_R/TAZ_peak/TAZ_peaks.narrowPeak"))
```

### `import()`

- From **rtracklayer** (Bioconductor)
- Reads genomic annotation files:
    - BED
    - narrowPeak
    - broadPeak
    - GTF, etc.

**It returns a GRanges object.**

---

### What is a `GRanges` object?

A `GRanges` object is:

- A collection of genomic intervals
- Each interval has:
    - chromosome
    - start
    - end
    - strand (optional)
    - metadata columns (score, signalValue, pValue, etc.)

Conceptually:

> “**A structured list of genomic regions that R knows how to compare spatially**.”
> 

That spatial awareness is what makes overlap calculations possible.
